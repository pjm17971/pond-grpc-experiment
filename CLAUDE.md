# pond-grpc-experiment — agent guide

This repository is one of three coordinating workstreams in the pond-ts ecosystem
(the others are the `pond-ts` library itself and the `pond-web` dashboard). Each
has its own agent. Feedback flows between agents through the human (Peter), so
identifying who wrote a given GitHub artifact matters.

## Agent identity in PRs and comments

Going forward, every PR description, PR review body, and PR comment authored by
this agent **must** start with an italicised attribution header on its own line:

```
_Posted by the gRPC experiment agent (Claude)_
```

When the comment plays a specific role in the PR/review workflow, append a short
em-dash suffix so a human skimming the thread can tell at a glance what they're
reading:

| Role                                              | Header                                                              |
| ------------------------------------------------- | ------------------------------------------------------------------- |
| Default PR description / general comment          | `_Posted by the gRPC experiment agent (Claude)_`                    |
| Layer-2 adversarial review (fresh agent)          | `_Posted by the gRPC experiment agent (Claude) — adversarial review_` |
| Author's response to an adversarial review        | `_Posted by the gRPC experiment agent (Claude) — review response_`  |
| Friction-note submission to the library agent     | `_Posted by the gRPC experiment agent (Claude) — friction report_`  |
| End-of-milestone wrap-up posted on a tracking PR  | `_Posted by the gRPC experiment agent (Claude) — milestone report_` |

Notes:

- The header appears once, at the very top of the body, followed by a blank line
  and then the actual content. No closing signature.
- The `Co-Authored-By: Claude …` git trailer on commits is unchanged — that
  attribution lives at the commit level, this header lives at the
  PR/review/comment level.
- This is going-forward only; do not retroactively edit existing PRs or
  comments to add the header.
- The canonical version of this convention lives in `pond-ts`'s `CLAUDE.md`
  under "Multi-agent experiments and the feedback model → Agent identity in
  PRs and comments". If the two ever drift, that one wins.

## Project shape (pointers, not duplication)

- `friction-notes/` — accumulated API-friction observations per milestone; feeds
  the M5 RFC sweep.
- `packages/aggregator/BENCH.md` — canonical throughput numbers; regenerated by
  `pnpm bench:full`.
- `packages/*/README.md` (where present) and inline source docstrings are the
  source of truth for how each tier works.
